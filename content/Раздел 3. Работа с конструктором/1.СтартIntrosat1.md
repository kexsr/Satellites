## Первая программа и общение с компьютером через UART

> **Зачем это нужно?**
> 
> Эта инструкция поможет вам "оживить" вашу плату BluePill и наладить с ней диалог. Вы научитесь:
> 
> 1. **Загружать программу** в микроконтроллер BluePill.
> 2. **Получать данные** от платы на компьютер — это основа для отладки и сбора информации с датчиков в будущих проектах.
>    
>    Эти навыки — фундамент для создания любых умных устройств: от мигалки до системы мониторинга на спутнике.


### Часть 1: Первая программа на BluePill

**Цель:** Заставить встроенный светодиод на плате BluePill мигать. Это тест, подтверждающий, что плата жива и выполняет ваши команды.

#### Шаг 1. Программирование

**Задача:** Напишите программу для мигания встроенного светодиода в среде разработки (Arduino IDE), загрузите его на плату BluePill через программатор ST-Link, используя схему подключения из таблицы в следующем шаге. 

>[!info]- Пример программы
>
> ```
> void setup() {
> pinMode(PC13, OUTPUT);
> }
> void loop() {
> digitalWrite(PC13, LOW);
> delay(1000);
> digitalWrite(PC13, HIGH);
> delay(1000);
> }
> ```

#### Шаг 2. Сборка

Программатор подсоединяется к Blue Pill по четырем проводам, выделенным в таблице:

|ST-Link V2|BluePill|
|---|---|
|![](https://orbita.education/images/13267)|![](https://orbita.education/images/13268)|
|SWCLK >---|---> SCK (SWCLK, CLK)|
|SWDIO >---|---> DIO (SWDIO, IO)|
|GND >---|---> GND|
|3.3V >---|---> 3.3V|

После подключения программатора станет возможна загрузка программы в плату.

### Часть 2: Настройка UART

**Цель:** Научить плату общаться с компьютером. UART — это простой и надежный способ передачи данных, который используется повсеместно: от отладки до обмена данными между модулями спутника.

> Более подробно ознакомиться с UART вы можете самостоятельно, например, изучив данный материал: [UART - Универсальный Асинхронный Приёмопередатчик](https://wiki.iarduino.ru/page/serial-protocols-uart/)

#### Шаг 1. Сборка

Чтобы воспользоваться UART на плате BluePill, необходимо подключить к плате конвертер usb-uart. Выводы микроконтроллера TX и RX расположены на обратной стороне материнской платы.

![](https://orbita.education/images/7598)

Схема соединения конвертера к плате:

|Название пина на МК|Название пина на конвертере|
|---|---|
|Tx|RXD|
|Rx|TXD|
|Gnd|GND|
|3.3V|3.3V|

После подключения конвертера к плате необходимо также подключить его к usb-порту вашего ПК посредством USB кабеля.

#### Шаг 2. Программирование

**Задача:** Напишите программу для вывода любой фразы в монитор порта в среде разработки (Arduino IDE), загрузите его на плату BluePill через программатор ST-Link. 

> При инициализации работы с монитором порта, установите скорость передачи данных **115200 бод** (бит/с), а также конфигурацию (второй параметр функции) **SERIAL_8E1**.
 
>[!info]- Расшифровка конфигурации SERIAL_8E1:
> - общая **длина** передаваемых данных в секунду - 8 бит;
> - проверяем данные на наличие ошибки, подставляя девятый **бит четности `E`** так, чтобы общее количество единиц было четным;
>   
>   > **Пример:** Вы передаете букву 'A': `01000001` (здесь две единицы — уже четное количество). С битом четности `E` будет: `01000001` + `0` (дописываем 0, чтобы количество единиц осталось четным = 2).
>   > 
>   > Если при передаче один бит "перевернулся" (из-за помех), получатель это заметит и может запросить повторную отправку. Это критически важно для космических и промышленных систем.
>   > 
> - После 8 бит данных и бита четности отправляется **1 стоповый бит**, который говорит: "Всё, данных больше нет, можно готовиться к следующему символу".

>[!info]- Пример программы
>
> ```
> void setup() {
> // Инициализируем работу с монитором порта
> Serial.begin(115200, SERIAL_8E1);
> }
> void loop() {
> // Отправляем строку текста в UART
> Serial.println("Hello World!");
> delay(1000);
> }
> ```

#### Шаг 3: Загрузите программу и откройте монитор порта

Чтобы увидеть результат работы придерживайтесь следующего алгоритма:

1. **Загрузите программу**, выбрав порт, в который подключена плата BluePill через ST-Link.
   
2. **Поменяйте порт** на тот, в котором находится конвертер USB-UART для считывания данных.
   
3. **Откройте монитор порта** в Arduino IDE:
    
    - Нажмите кнопку с **лупой** в правом верхнем углу
        
    - Или через горячие клавиши: _Ctrl + Shift + M_
    
4. **Настройте монитор порта:**
    
    - Скорость: **115200**
        
    - Перевод строки: **NL & CR** (или "Новая строка")
    
5. **Увидьте результат!** Каждую секунду в окне будет появляться строка с вашим текстом.

## Что такое датчик?

Датчик — это устройство, которое **преобразует физические явления** (свет, температуру, давление, движение) в **электрические сигналы**. Он служит «органами чувств» для любой автоматической системы, позволяя ей «понимать», что происходит во внешнем мире.

#### Как устроен датчик?

Любой датчик состоит из двух ключевых частей:

1. **Чувствительный элемент (detector)** — непосредственно «ощущает» внешнее воздействие (например, нагревается, изгибается).
    
2. **Преобразователь (transducer)** — превращает реакцию чувствительного элемента в **электрический сигнал** (изменение напряжения или тока), который уже можно измерить.
    

#### Язык сигналов: аналоговый и цифровой

Датчики общаются с другими устройствами с помощью сигналов. У любого сигнала есть две основные характеристики:

- **Амплитуда** — максимальное значение сигнала (например, уровень напряжения).
    
- **Частота** — сколько раз сигнал повторяется за секунду.
    

Сигналы бывают двух принципиально разных типов:

|Тип сигнала|Как работает|Пример из жизни|
|---|---|---|
|**Аналоговый**|**Непрерывный** сигнал, плавно меняющийся во времени. Может принимать бесконечное множество значений.|Звук, естественный свет, температура в комнате.|
|**Цифровой**|**Дискретный** сигнал, имеющий только два четких состояния: «ВКЛ» (HIGH, 1) и «ВЫКЛ» (LOW, 0).|Данные в компьютере, команды от кнопки.|

Чтобы компьютер или контроллер (например, Arduino) мог понять аналоговый сигнал, используется **аналогово-цифровой преобразователь (АЦП)**. Он «оцифровывает» непрерывный сигнал, разбивая его на дискретные значения.

![[../../Pasted image 20251210205930.png]]

>[!example]- Простой пример: потенциометр (переменный резистор)
> 
> Это самый наглядный пример датчика. Его принцип действия лежит в основе многих других сенсоров.
> 
> - **Что это?** Ручка или ползунок, изменяющие **сопротивление** в цепи.
> - **Как работает?** При повороте ручки длина токопроводящего пути меняется. Согласно **закону Ома**, это приводит к изменению **напряжения** на выходе.
>   
>   **Принципиальная схема работы:**  
>   
>   Поворот ручки потенциометра → Изменение сопротивления → Изменение напряжения (аналоговый сигнал) → Преобразование в цифровой код (АЦП) → Управление яркостью светодиода.

**Вывод:** Датчики — это фундамент автоматики и робототехники. Они переводят язык физического мира (аналоговые величины) на язык, понятный электронным системам (электрические сигналы), открывая путь для создания «умных» устройств — от простого регулятора освещения до систем ориентации космического спутника.

## Создаём систему с «глазами» и «мозгом»

> **Зачем это нужно?**  
> 
> Сегодня вы соберёте и запрограммируете своё первое «умное» устройство. Вы научитесь:
> 
> 1. **Считывать информацию** из внешнего мира с помощью датчика (потенциометр).
> 2. **Обрабатывать её** с помощью микроконтроллера (мозга системы).
> 3. **Управлять исполнительным устройством** (светодиодом) на основе полученных данных.
> 
> Это основа **любой** автоматики: от регулятора яркости лампы до системы ориентации спутника, которая по датчикам освещённости находит Солнце, чтобы направить к нему солнечные батареи. Успех этой сборки — ваш первый шаг к пониманию того, как работают сложные космические аппараты.


### Часть 1: «Регулятор яркости» (Работа с потенциометром)

**Цель:** Создать цепь, где поворот ручки плавно меняет яркость светодиода.

#### Шаг 1. Схемотехника

|Описание|Изображение|
|---|---|
|1. Соедините материнскую плату и плату BluePill, как показано на фото. _ВАЖНО! Разъем microusb на плате BluePill должен находиться над надписью «USB SIDE»._|![](https://orbita.education/images/12632)|
|2. Соберите на макетной плате простейшую цепь, состоящую из светодиода, резистора и потенциометра для регулирования напряжения.|![](https://orbita.education/images/12633)|

#### Шаг 2. Программирование

**Задача:** Напишите программу для правильной работы потенциометра в среде разработки (Arduino IDE), загрузите его на плату BluePill через программатор ST-Link. 

Программа должна:

- получать данные (значение напряжения) потенциометра;
- преобразовывать полученные данные в параметры для светодиода;
- передавать результат на светодиод.

**Проверьте работу:** поворачивайте ручку потенциометра. Яркость светодиода должна плавно меняться.

 >[!info]- Пример программы
 >
 >``` #define led PB11
 >#define pot PA0
 >void setup() {
 >pinMode(led, OUTPUT);
 >pinMode(pot, INPUT);
 >}
 >void loop() {
 >//считывание значений с потенциометра

 >//в диапазон для светодиода
 >int y = map(x, 0, 1023, 0, 255);

 >}
 >```


### Часть 2: «Глаз спутника» (Работа с датчиком освещённости)

**Цель:** Научить систему измерять уровень света, как это делает спутник для поиска Солнца, используя [[../Теория/Датчик освещенности|датчик освещенности]].

#### Шаг 1. Схемотехника

Перед подключением датчика к плате убедитесь, что все переключатели стоят в нижнем положении. Такая конфигурация датчика соответствует определенному режиму работы. Для подключения датчика освещенности используем соединительные провода «мама-папа». 

Провода должны быть подключены следующим образом:

![](https://orbita.education/images/7646)

> - контакт «SCL» подключается к цифровому «pin» (например D9);
> - контакт «Vcc» подключается к контакту «3.3V»;
> - контакт «GND» подключается к контакту «GND».

#### Шаг 2. Программирование

**Задача:** Напишите программу, которая считывает длительность импульса с датчика и выводит его значение в монитор порта.

**Проверьте работу:** Используя конвертер usb-uart, открываем монитор порта для отображения результата работы программы. Результатом работы программы является вывод числовых значений от 0 до 1800, где 0 – наиболее интенсивное освещение, а 1800 – наименее интенсивное.

> Для считывания значения импульса используйте команду **`PulseIn(pin, state)`**, где:
> 
> - `pin` - это название пина, к которому подключен датчик,
> - `state` - тип ожидаемого сигнала (`HIGH` - переход c низкого на высокий или `LOW` - переход с высокого на низкий).


> [!info]- Пример программы
> 
>```
>#define light_sensor1 PB11
>void setup() {
>pinMode(light_sensor1, INPUT);
>//зададим настройки для работы с монитором порта
>Serial.begin(115200, SERIAL_8E1);
>}
>void loop() {
>//записываем значение сигнала в переменную light1
>long light1 = pulseIn(light_sensor1, HIGH);
>//отправляем полученное значение в монитор порта
>Serial.println(light1);
>}
>```
