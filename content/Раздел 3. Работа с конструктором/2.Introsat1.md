## 🚀 Сборка и программирование системы ориентации солнечной батареи наноспутника

---

### 📜Часть 1: Теоретическая основа

#### Исполнительные устройства

В системах автоматики (как в нашем спутнике) недостаточно просто измерить данные. Нужно уметь на них реагировать. Для этого служат **исполнительные устройства (ИУ)**.

**Что это?** Механизм, который по команде от управляющего блока (нашей платы) физически воздействует на систему.

**В нашем проекте:** Исполнительное устройство — **шаговый двигатель**, который будет поворачивать панель.

**Примеры в космосе:** двигатели коррекции орбиты, маховики для стабилизации, **механизмы поворота солнечных батарей**.

#### Шаговый двигатель

**Главное преимущество:** Высокая точность позиционирования. Двигатель не вращается непрерывно, а поворачивается на строго заданный угол («шаг»).  
**Устройство:** Внутри находится магнитный ротор, окруженный несколькими электромагнитными катушками (обмотками).

**Принцип работы шагового двигателя** ([видео по ссылке](https://rutube.ru/video/67259dbc9663b33393c98d77695e1a64/)):

1. На одну из обмоток подается ток — создается магнитное поле.
2. Магнитный ротор поворачивается, стремясь совместиться с этим полем.
3. Ток переключается на следующую обмотку — ротор делает новый шаг.  
    _Последовательное переключение обмоток заставляет ротор вращаться._


![[../images/Slide 16_9 - 10.png]]


Существует несколько режимов управления двигателем. Рассмотрим режим, в котором ток подается на две соседние обмотки (полношаговый режим):


![[../images/Slide 16_9 - 11.png]]

### 💭 Логика работы стенда

1. Четыре датчика освещенности, смотрящие в разные стороны (под 90°), измеряют уровень света.
2. Микроконтроллер (плата Blue Pill) сравнивает показания с датчиков.
3. Определяется направление на **наиболее яркий источник света**.
4. Контроллер дает команду **шаговому двигателю** повернуть солнечную батарею в этом направлении.

### 👨‍🔧 Часть 2: Практическая сборка

**1. Сборка механической части**

1. Соберите каркас из рельсов и перекладин, скрепляя винтами **М2.5х12**.
    
2. Закрепите **шаговый двигатель** на нижней панели (вал должен проходить в центральное отверстие).
    
3. Установите **датчики освещенности** на боковые панели винтами **М2.5х6**.


![[../images/Slide 16_9 - 12.png]]


**2. Сборка электронной части**

1. Установите плату **Blue Pill** на материнскую плату. **ВАЖНО:** Разъем MicroUSB должен быть над надписью «USB SIDE».
    
2. Установите модуль питания с аккумулятором.
    
3. Закрепите собранный «сэндвич» плат на верхней панели с помощью стоек **PCHSN-11**.
    
4. **Подключите двигатель:**
    
    - Входы `I1, I2, I3, I4` → пины `PA15, PB3, PB4, PB5`.
        
    - Выход `SW1` → пин `PC15`.
        
    - Подайте **5В** от платы на разъем питания двигателя.


![[../images/Slide 16_9 - 7.png]]


### 👨‍💻 Часть 3: Программирование шагового двигателя

Напишем новую программу. Добавим индикаторы для управления шаговым двигателем.

```
#define in1 PB5
#define in2 PB4
#define in3 PB3
#define in4 PA15
```

Подключим библиотеку для использования bluetooth-модуля.

```
#include <IS_Bluetooth.h>
```

>[!info] Библиотеку «IS_Bluetooth» и другие библиотеки можно скачать по [ссылке](https://github.com/Obu-IntroSat/IntroSatLib).

Далее внутри функции setup зададим начальные настройки.

```
  Serial.begin(115200, SERIAL_8E1);
```

А также пины для управления шаговым двигателем.

```
  pinMode(in1, OUTPUT);
  pinMode(in2, OUTPUT);
  pinMode(in3, OUTPUT);
  pinMode(in4, OUTPUT);
```

Определим еще один дополнительный пин для определения начального положения шагового двигателя.

```
 pinMode(PC15, INPUT);
```

Теперь напишем программу управления, которая позволит управлять шаговым двигателем. 

На схеме ниже представлен режим управления двигателем. Красным цветом показаны обмотки двигателя, задающее южный полюс магнитного поля, а синим - северный. Смещая полярность обмоток на одну на каждом шаге по часовой стрелке, можно осуществить полный цикл поворота магнита двигателя, который повернет на небольшой угол и вал шагового двигателя и будет соответствовать, так называемому, шагу двигателя.


![](https://orbita.education/images/12766)

Напишем перед функцией _loop()_ вспомогательную функцию, которая будет задавать поворот на заданный угол по часовой стрелке для каждого шага двигателя.


![](https://orbita.education/images/12767)

**Код, соответствующий Шагу 1:**

```
// функция, задающая поворот на заданный угол по часовой стрелке
void right_rotation(int alpha)
{
  for (int i = 1; i <= ceil(alpha * 52 / 36); i++)
  { 
    // подача напряжения на 3 и 4 пин 
    digitalWrite( in1, LOW );
    digitalWrite( in2, LOW );
    digitalWrite( in3, HIGH );
    digitalWrite( in4, HIGH );
    delay(10);

```

>[!info] Цикл for
>
>Здесь используется цикл `for` с рассчитываемым значением счетчика `i`. Дело в том, что у шагового двигателя 520 шагов приходится на полный оборот. А нам удобнее управлять им, задавая угол поворота в градусах. Поэтому используем преобразование `ceil(alpha * 52 / 36)`, где alpha - угол, на который мы хотим повернуть вал, а 520/360 = 52/36 - преобразование шагов двигателя в поворот на один градус. 
>
>Представленный на картинке цикл соответствует одному полному шагу двигателя, а нам соответственно нужно будет сделать `alpha * 52 / 36` шагов, чтобы повернуться на угол `alpha`. 
>
>Функция `ceil` используется для округления полученного значения до целого числа (цикл `for` использует счетчик в виде целого числа).

**Далее запишем код для Шага 2.**

![](https://orbita.education/images/12768)

```
    // подача напряжения на 4 и 1 пин, чтобы осуществить проворот 
    digitalWrite( in1, HIGH );
    digitalWrite( in2, LOW );
    digitalWrite( in3, LOW );
    digitalWrite( in4, HIGH );
    delay(10);

```

**Аналогично для Шага 3.**

![](https://orbita.education/images/12769)

```
    // подача напряжения на 1 и 2 пин, чтобы осуществить проворот 
    digitalWrite( in1, HIGH );
    digitalWrite( in2, HIGH );
    digitalWrite( in3, LOW );
    digitalWrite( in4, LOW );
    delay(10);
```

**И также для Шага 4.** 

После четвертого шага цикл повторяется, возвращаясь в положение Шага 1.

![](https://orbita.education/images/12770)

```
    // подача напряжения на 2 и 3 пин, полный цикл завершится, если снова подать напряжение на 3 и 4 пин 
    digitalWrite( in1, LOW );
    digitalWrite( in2, HIGH );
    digitalWrite( in3, HIGH );
    digitalWrite( in4, LOW );
    delay(10);
  }
}

```

> [!info] Задачка
> 
> Аналогично можно прописать функцию для поворота против часовой стрелки, записав управление пинами в обратном порядке.

Теперь можно проверить работу реализованных функций управления, дописав их вызов в функции loop.

```
void loop() {

//код для прошивки через bluetooth-модуль
if (Serial.available()) {
    char command = Serial.read();
    if (command == 'b') {
      enter_bootloader();
    }
  }

  right_rotation(45);
  delay(1000);
  left_rotation(45);
  delay(1000);
}

```

Если шаговый двигатель подключен правильно, а код для него корректно вписан в программу, то вал двигателя будет вращаться туда-обратно на указанный угол.



### 👨‍🔧 Часть 4: Подключение датчиков освещенности

1. Закрепите верхнюю панель на раме с помощью винтов М2.5х6 с потайной головкой
2. К каждому датчику подсоедините три провода к контактам GND, SCL и Vcc, как описывалось на предыдущем занятии.
3. Каждый контакт SCL с датчика подсоедините к цифровому пину, например, PA8, PB11, PB10, PA11. Установите соответствие между гранью и пином, чтобы понимать расположение датчиков на кубсате.
4. Закрепите боковые панели на раме с помощью винтов М2.5х6 с потайной головкой. 3.5. Все контакты датчиков GND подсоедините к линейке пинов GND на плате прототипирования, а пины Vcc - к 3,3V.

###  ✅ Часть 7: Программирование стенда солнечной батареи

Важным условием работы стенда солнечной батареи будет проектирование оптимального алгоритма работы системы датчиков. Система состоит из четырех датчиков, расположенных на сторонах куба. Предполагая, что источник освещения направлен перпендикулярно к стороне, а значит, и расположенному на ней датчику, можно создать систему управления, проверяющую состояние каждого датчика через некоторый промежуток времени и поворачивающую солнечную батарею параллельно наиболее освещенному датчику.

В результате у солнечной батареи будет **четыре состояния**:

- поворот на 0 градусов (параллельно датчику 1);
- поворот на 90 градусов по часовой стрелке (параллельно датчику 2);
- поворот на 180 градусов по часовой стрелке (параллельно датчику 3);
- поворот на 270 градусов по часовой стрелке (параллельно датчику 4)»

![](https://orbita.education/images/7860)

#### Шаг 1. Блок-схема работы стенда с солнечной батареей

> [!info]- Что такое блок схема?
> 
> **Блок-схема** — это простой графический способ объяснить, как работает программа.
> 
> ![[../images/блоксхема.jpg]]

Перед тем как браться за реализацию сложного алгоритма необходимо составить его графическое представление. 

#### Шаг 2. Составление алгоритма всей программы

Краткий алгоритм решения задачи будет такой:

1. Прописываем названия для пинов, подключаем библиотеку для работы с bluetooth и задаем начальные настройки ([[#👨‍💻 Часть 3 Программирование шагового двигателя|из предыдущей части урока]]).

2. Создаем функции поворота шагового двигателя по часовой стрелки `right_rotation` и против часовой стрелки `left_rotation`.
   
3. Начинаем программировать в функции `loop()`: прошиваем bluetooth-модуль.
   
   >[!example]- Код для прошивки
   >
   >```
   >if (Serial.available()) {
   >    char command = Serial.read();
   >    if (command == 'b') {
   > 		   enter_bootloader();
   >    }
   > }
   > ```

4. Реализуем поворот двигателя в начальное положение.
   
   >[!example]- Код
  >
  >```
  >//инициализируем переменную логического типа 
  >//(когда достигается 0 градусов, она выводит False и выходит из цикла)
  >bool cont = digitalRead(PC15);
  >while (cont) {
  >	left_rotation(5);
  >	cont = digitalRead(PC15);
  >}
  >```

5. Снимаем показания с датчиков освещенности, как в [[#👨‍💻 Часть 5 Программирование датчиков освещенности|предыдущей части урока]] .
   
6. Осуществляем управление двигателем, основываясь на составленной блок-схеме.
   
   ![[../images/блоксхема.jpg]]>[!example]- Код
   >
   >```
   >// если максимум света на 1-м датчике
   >if (light1 < light2 && light1 < light3 && light1 < light4) {
   >	right_rotation(0);
   >}
   > // если максимум света на 2-м датчике
   > else if (light2 < light1 && light2 < light3 && light2 < light4) {
   >	right_rotation(90);
   >}
   >// если максимум света на 3-м датчике
   > else if (light3 < light1 && light3 < light2 && light3 < light4) {
   >	right_rotation(180);
   >}
   >// если максимум света на 4-м датчике
   > else if (light4 < light1 && light4 < light2 && light4 < light3) {
   >	right_rotation(270);
   >}
   >```
   

>[!info] Важный момент
>
>Из-за конструктивных особенностей шагового двигателя, возможно, понадобится некоторая поправка поворота двигателя на 10-45°. 
>
>Для того, чтобы скорректировать эту поправку, необходимо добавить ее к повороту на определенный угол, т.е., если при сборке стенда получается, что солнечная батарея встает под углом в 15° к грани, нужно поворачивать батарею не на 0°, 90°, 180° и 270°, а на 15°, 105°, 195° и 285°. Необходимую корректировку легко внести в алгоритм.


#### Шаг 3. Проверка работы программы

Чтобы получить на солнечной батарее ток, значение которого отобразится на экране амперметра, необходимо **установить источник света с лампой накаливания на расстоянии 10 см** от грани собранного куба. Чтобы проверить работу алгоритма, можно несколько раз изменить положение источника света. Каждый раз, при смене положения источника света, солнечная батарея должна стремиться повернуться к нему.

### ⭐ Дополнительное задание

В качестве дополнительного задания можно усовершенствовать алгоритм управления, введя в него отсчет угла поворота. Это позволит не возвращаться каждый раз в нулевое положение, а просто доворачиваться на нужный угол.

При написании своего алгоритма учитывайте, что поворачиваться от нулевого положения влево двигатель не может, так как там стоит ограничитель поворота. Поэтому отсчитывайте угол от нулевого положения и вращайте двигатель вправо. Если же необходимо повернуть двигатель влево, то учитывайте, что задаваемый угол поворота не должен превышать текущий.
